# Contributing to Ruvia Cosmetics

We maintain high standards for all code contributions to the Ruvia Cosmetics digital infrastructure. Please follow these guidelines to ensure consistency and quality.

## Code Standards
- **Framework**: Next.js (App Router)
- **Styling**: Tailwind CSS 4.0
- **Animations**: GSAP 3.x
- **Components**: Functional components only, with strict prop-type validation (if applicable).

## Commit Messages
We follow the [Conventional Commits](https://www.conventionalcommits.org/) specification:
- `feat:` for new features
- `fix:` for bug fixes
- `docs:` for documentation changes
- `refactor:` for code changes that neither fix a bug nor add a feature
- `perf:` for performance improvements

## Pull Request Process
1. Ensure all linting checks pass.
2. Update the README.md with details of changes to the interface or architecture.
3. The project must build successfully (`npm run build`) before submission.

## Clinical Integrity
Any changes to clinical descriptions, ingredient lists, or medical terminology must be approved by the lead pharmacist or head dermatologist.
